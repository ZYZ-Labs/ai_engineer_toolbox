# 第 12 章：构建发布与下一步

## 学习目标

- 创建桌面构建。
- 做发布前检查。
- 给试玩包命名和记录版本。
- 规划下一批 Unity 练习项目。

## 12.1 构建前检查

构建前确认：

1. Console 没有红色错误。
2. MainMenu 和 Main 都在 Build Settings。
3. 游戏可以开始、游玩、失败。
4. 分数和生命值显示正常。
5. 所有必要资源在 Assets 下。
6. 没有把 Build 目录提交到 Git。

## 12.2 创建构建

打开 Build Profiles 或 Build Settings：

1. 选择目标平台。
2. 确认场景列表。
3. 选择输出目录，例如项目外的 `UnityDodgeBuilds/`.
4. 点击 Build。

不要把构建输出直接放进 `Assets`。

## 12.3 版本命名

试玩包命名：

```txt
UnityDodgeStarter-v0.1.0-windows.zip
UnityDodgeStarter-v0.1.0-macos.zip
```

README 写清楚：

- Unity 版本：Unity 6.3 LTS。
- 操作方式。
- 目标平台。
- 已知问题。

## 12.5 不要过早复杂化

新手阶段先把小项目做完。等出现真实复杂度后，再引入事件系统、状态机、依赖注入、复杂架构。

优先改进：

- 小脚本。
- 清晰目录。
- Prefab 复用。
- 数据和逻辑分离。
- 每一步可运行验证。

## 本章练习

1. 构建一个桌面试玩包。
2. 在另一台机器或新目录运行。
3. 写 README。
4. 用 Git 打 tag `v0.1.0`。
5. 找一个新手试玩并记录反馈。

## 检查点

- 你能从空项目做到可试玩包。
- 你知道构建不是保存项目。
- 你有下一步练习路线。
